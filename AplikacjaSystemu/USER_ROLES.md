# User Roles and Groups

This document lists all the user roles (groups) defined in the Academic Management System and their responsibilities.

| Role Code | Full Name / Description | Responsibilities |
| :--- | :--- | :--- |
| **Employee** | **Academic / Administrative Employee** | Can initiate standard requests (e.g., Leave Requests). Most users belong to this group. |
| **HeadOU** | **Head of Organizational Unit** | **System Role**. Grants permission to approve requests for a unit. In the real world, this is a "Position" (Kierownik), but for the system to know *who* can click "Approve", they must have this Role. |
| **PD** | **Personnel Department (HR)** | Handles administrative verification, compliance checks, data entry into HR systems, and archiving documents. Involved in almost every process. |
| **KWE** | **Quaestor (Bursar) / Finance** | Verifies financial availability for requests involving costs (e.g., salary changes, employment conditions). |
| **Rector** | **Rector** | Highest decision-maker for academic matters and high-level approvals (e.g., Medals, critical employment changes). |
| **Chancellor** | **Chancellor** | Decision-maker for non-academic/administrative matters and financial approvals. |
| **PRK** | **Vice-Rector for Education** | Reviews applications related to educational staff and academic teachers (Education track). |
| **PRN** | **Vice-Rector for Science** | Reviews applications related to scientific staff and researchers (Science track). |
| **MPD** | **Marketing & Promotion Dept.** | Handles external processing of awards, medals, and distinctions. |
| **Admin** | **System Administrator** | Technical role. Has access to the Admin Panel to manage user accounts and assign roles. Does not participate in business processes directly. |
